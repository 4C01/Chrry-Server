# Chrry - 一只私人管家小猫咪

_外桂和友商发力了,谁说全栈MC开发不能做小玩意了_  

## 介绍与功能

Chrry 是一只可爱又实用的私人AI管家小猫咪，能够根据您的不同需求进行定制，帮助您处理日常任务、回答问题、提供灵感等。通过不同的会话配置，您可以拥有专属于各种场景的AI助手。

_目前大概只兼容 OpenAI格式的API(Siliconflow, Deepseek, ChatGPT, Qwen, GLM etc.), 正在写Gemini_  
本项目采用 [MIT 许可证](./LICENCE)  

[API Usage](./openapi.yml) (你可以把这个喂给AI,让AI来测试连接并写一个客户端)

## 快速入手

### 准备环境
- 一个可以联网的电脑
- 安装一个 Python3.8+ (我用的3.13)
- 可爱的小爪爪
- 一个 Chrry 的客户端(你现在看到的这个是服务端!~是管家的总管!~)

### 安装并启动
1. 拉下仓库,或者下载ZIP并解压
    ```bash
    git clone https://github.com/4C01/Chrry-Server.git
    ```
2. 打开你 解压/Clone 下来的文件夹,确保能看到一大堆文件(里面有个main.py的那种)
    ```bash
    cd Chrry-Server
    ```
3. 启动!~(快速点两下 start.bat 文件), Linux(租的服务器,装了Debian啊Centos什么的)请:
    ```bash
    chmod +x ./start.sh
    bash ./start.sh
    ```

### 使用方法
启动后会输出一些乱七八糟的东西, 用鼠标滚轮或者右边的条翻找到最上面, 找到一个和下面的长得差不多的东西
```
新API密钥已生成: bakabaka-zako-nyaa-neko-114514191981,保存至data/api_key.json
```
_这里的_ ```bakabaka-zako-nyaa-neko-114514191981``` _是我乱编的www 你需要看你的输出里真的API密钥是什么_  

如果Key泄漏了,删除data下的api_key.json就行  

复制粘贴一个这个 API Key 并放入客户端 (去看客户端的教程不要看我啦aaa~)  

如果需要更改端口(5000被占用了的话), 请直接更改main.py最后一行的```port=5000```中的5000(是的, 我懒得写了, 有好心人的话帮我写个config吧qwq但是没什么必要)  

剩下的就是客户端的事情啦~ 因为大部分事情都可以在客户端操作(有个好看的页面总比对着黑框乱敲curl好)
